1. Выведите список всех файлов в папке html

ls -al /var/www/html # необходимости переходить в каталог для просмотра его содержимого нет. Дополнительный параметр -al покажет все файлы в каталоге (включая скрытые)

2. Найдите в файле access.log все упоминания строки "404"

grep -o '404' /var/logs/access.log # дополнительный параметр -о выведет только совпавший кусок строки 
# также можно использовать grep -c '404'  /var/logs/access.log для подсчета совпадений

3. На основании файла 404.html сделайте файл 403.html заменив в его содержимом все упоминания 404 на 403

cd/var/www/html/ # переходим в нужный каталог
touch 403.html # создадим новый файл-заготовку
cat 404.html |replace "404" "403" > 403.html # выполним копирование содержимого файла 404.html в файл 403.html с автозаменой

4. Модифицируйте права доступа и владельца файла index.html следующим образом: Текущий пользователь должен иметь права на чтение/запись Группа www-data на чтение Все остальные не имеют доступа к файлу

sudo chown user:www-data cd/var/www/html/index.html # меняем владельца и задаем группу 
cd/var/www/html/ # переходим в нужный каталог
chmod 640 index.html # 6 - чтение(4) и запись (2) для владельца, 4 - чтение(4) для группы, 0 - нет доступа для всех остальных

5. Сделайте символическую ссылку на файл access.log в папке html.

cd/var/www/html/ # переходим в нужный каталог
ln -s /var/logs/access.log newLink # создаем ссылку, newLink - любое имя для создаваемой ссылки

6. Выведите на экран все файлы в папке var

ls -R /var # дополнительный параметр R (recursive) выводит содержимое всех подкаталогов рекурсивным образом

7. Создайте новый файл 503.html в папке html с следующим содержимым:

cd/var/www/html/ # переходим в нужный каталог
touch 503.html # создадим новый файл
echo "<html>\n\t<head>\n\t\t<title>503</title>\n\t</head>\n</html>" > 503.html # запись в пустой файл поссредством команды echo, \n - перевод строки, \t - табуляция
